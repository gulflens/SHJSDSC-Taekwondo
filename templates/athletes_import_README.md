# Athlete Import Template

Open `athletes_import_template.csv` in Excel (or Numbers / Google Sheets), keep the
**first row exactly as-is** (it's the column header row the importer reads), then
add one athlete per row. Save the file as **CSV (Comma delimited)** before importing.

In the app: **Athletes tab → toolbar → Import (`square.and.arrow.down`)** → pick the
CSV. The importer pre-loads branches and coaches, resolves codes/names to IDs, and
calls the same `repository.upsert(_:)` path the Add Athlete form uses — so writes
flow into Supabase and respect the live schema.

## Columns

### Required
| Column | Notes |
|---|---|
| `full_name` | English name |
| `full_name_ar` | Arabic name |
| `date_of_birth` | `YYYY-MM-DD` |
| `gender` | `male` or `female` |
| `branch_code` | The branch's `code` field (e.g. `BR-NAS`, `BR-NOUF`, `BR-IND18`, `BR-RAH`). The importer rejects rows where the code doesn't match an existing branch. |
| `joined_at` | `YYYY-MM-DD` |
| `weight_kg` | Number |
| `current_belt_color` | `white` `yellow` `green` `blue` `red` `black` |
| `current_belt_kind` | `gup` `poom` `dan` |
| `current_belt_number` | Integer (10 = white, 1 = red, etc.) |
| `status` | `active` `competitionTeam` `readyToGrade` `watch` `rest` |

### Optional (leave blank if unknown)
| Column | Notes |
|---|---|
| `nationality` | ISO-2 code, default `AE` if blank |
| `emirates_id` | Format: `784-YYYY-NNNNNNN-N` |
| `passport_number` | |
| `blood_type` | `A+` `A-` `B+` `B-` `AB+` `AB-` `O+` `O-` |
| `federation_licence_number` | |
| `school` | |
| `height_cm` | Number |
| `allergies` | Semicolon-separated list (e.g. `peanuts;dairy`) |
| `medical_conditions` | Semicolon-separated list |
| `medications` | Semicolon-separated list |
| `fit_to_train` | `true` or `false` (default `true` if blank) |
| `dominant_stance` | `orthodox` `southpaw` `switchStance` |
| `poomsae_syllabus` | e.g. `taegeuk-3` |
| `kyorugi_tier` | `recreational` `competitive` `elite` |
| `primary_coach_full_name` | Must match an existing coach's `full_name` exactly. Blank = no coach assigned. |

## Server-allocated fields (don't include)
- `id` — generated by Postgres
- `member_number` — allocated lifetime-unique by `next_member_number()` RPC
- `avatar_seed` — derived from the first word of `full_name`


## What the importer does on errors
- A row with an invalid required field is **skipped** (not silently inserted with bad data).
- A row that references a non-existent `branch_code` or `primary_coach_full_name` is **skipped**.
- After the run you'll see a summary: `imported`, `skipped`, plus the row numbers + reasons for each skip — you can fix the CSV and re-import.
- Re-importing the same row creates **another athlete** (with a new server-allocated `member_number`); the importer does not de-duplicate by name. Edit existing athletes in the app instead.
